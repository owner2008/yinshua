# 前端主题与移动端验收说明

## 目标

本次前端改造的目标不是单次换皮，而是建立一套可持续维护的主题系统，让 H5 前台与微信小程序在视觉语言、层级结构和关键交互上保持一致。

当前已覆盖：

- H5 前台：`apps/client`
- 微信小程序：`apps/miniprogram`

## 当前主题方向

默认主题采用“深色底 + 金色强调 + 玻璃感卡片”的高级感方向，核心特点如下：

- 背景：深色渐变与轻微光斑，避免平铺纯色背景
- 卡片：半透明深色面板，配合柔和边框和阴影
- 强调色：金色系，用于主按钮、主题切换、关键标签
- 信息层级：标题高对比，正文偏柔和，说明文字进一步弱化
- 转化路径：首屏、产品、详情、报价页都保留明确入口

## H5 主题结构

H5 主题入口在：

- `apps/client/src/App.tsx`
- `apps/client/src/styles.css`

### 主题切换机制

H5 使用 `data-theme` 驱动整站样式切换。

当前主题值：

- `graphite`
- `ivory`
- `forest`

切换逻辑：

1. 在 `App.tsx` 中读取 `localStorage`
2. 将主题写入 `document.documentElement.dataset.theme`
3. 在 `styles.css` 中通过 `:root[data-theme='xxx']` 覆盖变量

### 样式维护原则

后续新增页面时，优先使用已有变量，不要直接写死颜色。

优先使用的变量分类：

- 背景：`--bg` `--bg-soft` `--bg-elevated`
- 面板：`--panel` `--panel-strong` `--card` `--card-hover`
- 文本：`--text` `--text-muted` `--text-soft`
- 线框：`--line` `--line-strong`
- 品牌强调：`--accent` `--accent-strong` `--accent-contrast` `--accent-soft`
- 阴影与圆角：`--shadow-lg` `--shadow-md` `--radius-xl` `--radius-lg` `--radius-md`

### 新增主题时建议步骤

1. 在 `App.tsx` 的 `themeOptions` 中增加新主题
2. 在 `styles.css` 中新增对应 `:root[data-theme='新主题']`
3. 只改 token，不直接改具体页面 class
4. 检查首页、产品列表、报价、详情、会员页是否都仍然可读

## 小程序主题结构

小程序主题入口在：

- `apps/miniprogram/app.json`
- `apps/miniprogram/app.wxss`

### 当前做法

小程序没有像 H5 一样做运行时多主题切换，而是先统一成一套稳定的品牌主题。

全局控制点包括：

- 页面底色
- 卡片背景
- 文本颜色
- 按钮样式
- 表单输入框
- tabBar 颜色
- navigationBar 颜色

页面级样式在以下目录分别补充：

- `pages/index`
- `pages/product-list`
- `pages/product-detail`
- `pages/quote`
- `pages/history`
- `pages/member`

### 后续如果要做小程序多主题

建议不要直接复制多套页面样式，而是：

1. 先在 `app.wxss` 抽一层更稳定的全局 token 命名
2. 页面尽量只消费公共 class 和少量页面专属布局 class
3. 如果以后要切换主题，可以考虑在页面 `data` 中增加主题 class，并按页面根节点挂载

## 首页布局规范

首页建议长期维持以下结构顺序：

1. 品牌首屏
2. 首页重点摘要
3. 产品分类入口
4. Banner 延展
5. 企业介绍
6. 设备能力展示
7. 热门产品
8. 最新上架

原因：

- 先讲品牌，再给入口
- 先建立信任，再引导转化
- 先给结构感，再给内容密度

如果未来继续扩展模块，优先加在“企业介绍”和“热门产品”之间，不建议打断首屏和分类入口的连续阅读。

## 组件风格约定

### 按钮

- 主按钮：品牌强调色渐变，承担主要转化
- 次按钮：透明或弱化背景，用于返回、刷新、辅助操作

### 卡片

- 大卡片用于首屏、企业介绍、设备能力等承载型内容
- 小卡片用于产品块、分类入口、摘要模块

### 文案

- 标题：短句，避免堆关键词
- 副文案：一句话说明价值或动作
- 空状态：说清“为什么空”和“接下来可以做什么”

## 小程序真机 / 开发者工具验收重点

建议在微信开发者工具中重点检查以下页面：

### 首页

- 首屏背景图是否被遮罩压得过暗
- 分类卡片是否在窄屏下仍然可读
- Banner 轮播文案是否被裁切
- 企业介绍和设备展示图片区是否撑满容器

### 产品列表

- 横向分类筛选是否顺畅
- 产品卡片两列在不同机型下是否挤压

### 报价页

- 横向产品选择是否易滑动
- 表单输入区域是否出现内容拥挤
- 主按钮和次按钮在小屏是否足够清晰
- 报价结果卡片金额是否有截断

### 产品详情

- 封面图和标题区是否对齐
- 图集两列是否有比例失衡
- 模板说明是否换行自然

### 会员中心

- 表单输入在窄屏下是否过高或过密
- 地址卡片长按是否容易触发
- “默认”徽标是否足够醒目

### 报价历史

- 卡片左右信息是否平衡
- 价格是否在长报价单号场景下被挤压

## 下一步建议

如果继续迭代，建议按下面优先级推进：

1. 在微信开发者工具里跑一轮真机尺寸预览，记录具体问题截图
2. 根据截图微调小程序的字号、卡片高度和按钮间距
3. 给 H5 和小程序补一份统一的组件命名表，减少后续样式分叉
4. 如果后续需要品牌升级，再考虑给小程序也做有限的主题切换能力

## 备注

本说明文档偏工程维护视角，主要用于后续继续迭代时快速定位主题入口和验收重点。
