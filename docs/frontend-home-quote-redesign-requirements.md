# H5 与小程序首页及报价体验重设计需求

更新时间：2026-04-27

## 需求背景

当前 H5 前端与微信小程序前端的首页内容展示、排版层级和视觉效果仍不够理想，需要参考主流不干胶 / 标签印刷网站重新设计首页布局、内容组织与报价入口。

本需求允许对当前前台显示设计、页面结构和布局进行较大调整，但必须保持已实现的后台内容管理能力可继续驱动前台展示。

## 设计目标

重设计目标：

- 布局合理，信息层级清晰。
- 使用方便，用户能快速找到产品、了解能力并进入报价。
- 不过分堆砌内容，首页避免把所有后台内容平铺展示。
- 效果美观、舒适、专业，符合不干胶印刷行业网站气质。
- 操作路径短，尤其要突出“产品详情 -> 在线报价”和“首页 -> 快速报价”。
- H5 与小程序保持一致的信息架构，但允许根据设备特性采用不同排版。

## 行业参考观察

参考方向来自当前主流标签 / 不干胶印刷网站与询价表单做法：

- 标签印刷网站通常将首屏聚焦在产品类型、核心卖点、交付速度、材料/工艺能力和报价入口，而不是在首屏堆满企业介绍。
- 成熟在线报价流程常采用分步式配置，例如先选标签类型、形状、尺寸，再选材料、数量和工艺。
- 卷标类产品经常需要补充贴标方式、出标方向、卷芯、每卷数量或卷外径等参数。
- 材料与表面处理是影响报价和用户决策的重要因素，需要以更直观的方式展示。
- 询价场景通常要求用户提供材质、尺寸、数量、贴标方式、印刷颜色/工艺、样品或设计文件。

## 首页重设计需求

### 信息架构

首页建议改为以下结构：

1. 首屏 Hero
   - 品牌名称 / logo。
   - 一句清晰价值主张。
   - 主 Banner 图。
   - 主按钮：立即报价。
   - 次按钮：查看产品。
   - 可展示 2-3 个关键卖点，例如快速打样、多材料可选、支持小批量/批量。

2. 快速产品入口
   - 以产品分类或应用场景作为入口。
   - 优先展示 4-8 个高频分类。
   - 避免过多 chip 堆砌，超过数量时进入产品列表页。

3. 热门产品 / 典型应用
   - 用卡片展示主推产品。
   - 每张卡片应包含产品图、名称、应用场景、一个报价入口。
   - 点击卡片进入产品详情，报价按钮进入报价页并携带产品参数。

4. 快速报价引导
   - 首页中部或靠前位置放置轻量报价入口。
   - 可采用“选择产品类型 + 尺寸 + 数量”的简化入口。
   - 完整配置仍进入报价页完成。

5. 材料与工艺能力
   - 展示核心材料：铜版纸、PET、PVC、热敏纸、合成纸、透明膜等。
   - 展示核心工艺：覆膜、模切、烫金、UV、白墨、防伪、可移胶等。
   - 以能力说明为主，不要把它做成密集参数表。

6. 企业与设备能力
   - 企业介绍和设备展示建议下移，作为增强信任的内容。
   - 控制篇幅，首页只展示摘要和 2-4 张代表图。
   - 详情内容后续可进入企业介绍页或能力详情页。

7. 服务流程
   - 设计为 3-5 步：选产品、填参数、上传文件/确认样稿、生产、发货。
   - 重点帮助用户理解如何下单和报价。

8. 底部联系与转化
   - 电话、微信、地址。
   - 再次提供“立即报价”和“联系客服”入口。

### H5 排版要求

- 首屏要有明确视觉焦点，不要同时展示过多卡片。
- 页面宽度、留白、字号层级要统一。
- 首页内容分区应有节奏：强视觉区、轻入口区、内容说明区、转化区。
- 产品卡片图片比例统一，标题和说明不应撑破卡片。
- 不使用过多装饰性渐变和大面积深色块，整体应干净、可信、专业。
- 移动端首屏不能被导航、状态文案和过长标题挤满。

### 小程序排版要求

- 首屏信息更克制，优先展示品牌、核心 Banner、立即报价。
- 首页分类入口使用 2 列或横向滑动，避免密集小按钮。
- 热门产品卡片适配手指点击区域。
- 报价入口必须在首页容易触达。
- 复杂说明内容折叠或下移，不在首屏密集展示。
- 需要考虑真机图片加载、长文本换行和按钮点击区域。

## 报价系统内容项补充需求

当前报价系统已有产品、模板、宽高、数量、材料、印刷方式、形状、工艺、打样、加急、客户类型等核心项。根据行业做法，建议评估补充以下内容项。

### 建议优先补充

1. 标签交付形式
   - 卷装。
   - 张装。
   - 单张裁切。
   - 折叠 / 风琴折。

2. 贴标方式
   - 手工贴标。
   - 自动贴标。
   - 半自动贴标。

3. 出标方向 / 卷标方向
   - 上出。
   - 下出。
   - 左出。
   - 右出。
   - 可选内卷 / 外卷。

4. 卷芯与分卷
   - 卷芯内径。
   - 每卷数量。
   - 最大卷外径。
   - 是否需要按数量分卷。

5. 文件与样稿
   - 是否已有设计文件。
   - 上传 AI / PDF / SVG / CDR / 图片。
   - 是否需要设计协助。
   - 是否需要电子样稿确认。
   - 是否需要实物打样。

6. 材料胶性 / 使用环境
   - 永久胶。
   - 可移胶。
   - 冷冻胶。
   - 强粘胶。
   - 户外 / 防水 / 耐油 / 耐高温 / 低温环境。

7. 表面处理
   - 哑膜。
   - 亮膜。
   - 哑油 / 光油。
   - 防刮。
   - 防水。
   - 白墨打底。

8. 印刷颜色与特殊印刷
   - 单黑。
   - 专色。
   - 四色。
   - 白墨。
   - 可变数据 / 条码 / 序列号。

9. 包装与发货
   - 包装方式。
   - 发货地区。
   - 期望交期。
   - 是否加急。

### 第一阶段建议做法

第一阶段不建议一次性把所有字段都纳入精确价格公式。建议分为三类：

1. 直接参与报价计算
   - 交付形式。
   - 贴标方式。
   - 出标方向。
   - 卷芯 / 每卷数量。
   - 表面处理。
   - 白墨。
   - 可变数据。
   - 加急。
   - 打样。

2. 先作为询价备注保存，后续再计价
   - 使用环境。
   - 包装要求。
   - 设计协助。
   - 特殊文件说明。
   - 期望交期。

3. 作为后台模板配置项
   - 某产品是否支持卷装 / 张装。
   - 某产品是否支持自动贴标参数。
   - 某材料是否支持白墨。
   - 某材料是否适合户外 / 冷冻 / 防水。
   - 某工艺是否可与指定材料组合。

## 报价页重设计需求

### 页面结构

报价页建议改为“分组表单 + 实时报价结果”的结构：

1. 产品与用途
   - 产品类型 / 具体产品。
   - 应用场景。
   - 客户类型。

2. 规格与数量
   - 宽度。
   - 高度。
   - 数量。
   - 形状。
   - 交付形式。

3. 材料与表面
   - 材料。
   - 胶性。
   - 表面处理。
   - 使用环境。

4. 印刷与工艺
   - 印刷方式。
   - 颜色 / 专色。
   - 工艺。
   - 白墨 / 可变数据。

5. 卷标与贴标
   - 贴标方式。
   - 出标方向。
   - 卷芯。
   - 每卷数量或卷外径。

6. 文件与交付
   - 是否已有设计文件。
   - 文件上传。
   - 是否打样。
   - 是否加急。
   - 发货地区 / 期望交期。

7. 报价结果
   - 总价。
   - 单价。
   - 材料费用。
   - 印刷费用。
   - 工艺费用。
   - 附加费用。
   - 保存报价。
   - 联系客服。
   - 再次修改参数。

### 交互要求

- 表单项按业务分组，不要把所有字段平铺。
- 根据产品模板动态显示字段，不支持的项不展示。
- 高级项默认折叠，例如卷芯、出标方向、可变数据。
- 报价结果固定在页面底部或侧边，方便用户随时看到结果。
- 报错提示要明确说明哪个参数不合法。
- 保存报价前必须保留完整输入快照。

## 后台配置影响

为支撑前台重设计，后台后续需要补充：

- 首页模块排序配置。
- 首页模块启停配置。
- 热门分类 / 热门产品选择。
- 材料特性标签。
- 工艺特性标签。
- 报价模板字段显隐配置。
- 报价模板默认值配置。
- 报价模板字段说明文案。

## 验收标准

### 首页验收

- H5 首页首屏清晰展示品牌、核心图、价值主张和报价入口。
- 小程序首页首屏不拥挤，按钮易点击。
- 用户可以在 2 次点击内从首页进入报价。
- 用户可以在 2 次点击内从产品卡片进入对应报价。
- 企业介绍和设备展示不喧宾夺主，但能增强信任感。
- 首页没有明显内容堆砌、文字溢出、按钮拥挤、卡片高度失衡。

### 报价验收

- 报价表单分组清晰。
- 用户能理解每一组参数的作用。
- 行业关键项已覆盖或可作为备注保存。
- 报价结果明细清楚。
- 保存报价后能在历史报价中查看完整参数。
- H5 与小程序核心流程一致。

## 实施建议

建议拆成两步做：

1. 先做信息架构与视觉重排
   - 重做 H5 首页。
   - 同步重做小程序首页。
   - 保持现有 API 不大改。

2. 再扩展报价字段
   - 先扩 DTO、数据库快照和前端表单。
   - 再逐步接入报价规则。
   - 无法立即计价的字段先保存为询价备注。
