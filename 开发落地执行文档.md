## 1. 项目目标

本项目建设一个面向客户的“不干胶印刷产品展示与参数化报价系统”，支持微信小程序、H5 与后台管理端。

第一期核心目标不是完整商城，也不是 ERP，而是先完成以下业务闭环：

**产品展示 -> 用户输入报价参数 -> 后端实时报价 -> 保存报价单 -> 历史报价可查 -> 后台可维护价格与规则 -> 新报价按新规则生效，旧报价保持不变**

第一期必须优先保证：

1. 报价规则可配置。
2. 报价计算在后端完成。
3. 每次报价保存完整快照。
4. 价格与规则修改可审计。
5. 产品展示与报价模板分离。
6. 报价与库存扣减解耦。

## 3. 总体架构

系统分为四层：

1. 前台展示与报价层：微信小程序、H5。
2. 业务 API 层：产品、会员、报价、库存、后台管理 API。
3. 核心业务引擎层：报价规则匹配、报价计算、价格快照、会员折扣、库存服务。
4. 数据存储层：MySQL、Redis、对象存储。

关键边界：

- 前端只收集参数、展示结果。
- 报价公式、价格参数、会员折扣、最低收费必须放在后端。
- 后台修改价格或规则后，只影响新报价。
- 历史报价必须通过快照保持不变。

### 6.2 报价计算流程

每次报价必须按以下顺序执行：

1. 校验产品、模板、尺寸、数量、材料、工艺是否合法。
2. 匹配当前生效的报价规则集。
3. 读取当前生效的材料价格、印刷价格、工艺价格、全局配置、会员系数。
4. 计算面积。
5. 计算材料成本。
6. 计算印刷成本。
7. 计算工艺成本。
8. 计算包装、运费、加急、打样等附加费用。
9. 汇总基础成本。
10. 应用利润系数。
11. 应用会员系数。
12. 应用最低收费。
13. 生成报价结果明细。
14. 保存报价单与报价快照。

### 6.3 核心公式

```text
面积 = widthMm * heightMm / 1,000,000

材料成本 = 面积 * 数量 * 材料单价 * 损耗率

印刷成本 = 数量 * 印刷单价 + 开机费

基础成本 = 材料成本 + 印刷成本 + 工艺成本 + 附加费用

销售价 = 基础成本 * 利润系数

会员价 = 销售价 * 会员系数

最终报价 = max(会员价 - 优惠金额, 最低收费)
```

### 6.4 报价入参 DTO

```ts
export class CreateQuoteDto {
  productId: number;
  productTemplateId: number;
  widthMm: number;
  heightMm: number;
  quantity: number;
  materialId: number;
  printMode: string;
  shapeType: string;
  processCodes: string[];
  laminationType?: string;
  hotStampAreaMm2?: number;
  uvAreaMm2?: number;
  isProofing?: boolean;
  isUrgent?: boolean;
  shippingRegionCode?: string;
  memberId?: number;
  customerType?: 'personal' | 'company';
}
```

### 6.5 必做接口

- `POST /api/quotes/calculate` 实时报价。
- `POST /api/quotes` 保存报价单。
- `GET /api/member/quotes` 我的报价列表。
- `GET /api/member/quotes/:id` 报价详情。
- `GET /admin/quotes` 后台报价列表。
- `GET /admin/quotes/:id` 后台报价详情。
- `GET /admin/quote-snapshots/:id` 查看报价快照。

### 6.6 验收标准

- 能用固定测试数据计算出正确报价。
- 后台修改价格后，新报价立即变化。
- 历史报价不随后台改价变化。
- 返回结果包含材料、印刷、工艺、附加费用、最终价格、单价。
- 非法尺寸、非法数量、非法材料、非法工艺会被拒绝。

## 7. P2：后台核心配置

后台优先服务于报价引擎，不先做复杂运营功能。

### 7.1 产品与报价模板

功能：

- 产品分类管理。
- 产品列表管理。
- 产品详情维护。
- 产品图集维护。
- 一个产品绑定多个报价模板。
- 模板配置尺寸范围、数量范围、可选材料、可选工艺、最低收费、默认损耗率。

验收标准：

- 产品和报价模板不是一对一强绑定。
- 用户端能根据产品获取可用报价模板。
- 模板能限制报价入参。

### 7.2 材料与材料价格

功能：

- 材料档案。
- 材料类型。
- 材料单位。
- 材料价格。
- 生效时间。
- 历史价格。

验收标准：

- 每条价格有 `effective_from`、`effective_to`、`is_current`。
- 改价后保留历史记录。
- 报价读取当前生效价格。

### 7.3 工艺与工艺价格

功能：

- 工艺档案。
- 收费模式。
- 单价。
- 最低收费。
- 开机费 / 刀模费 / 打样费。

收费模式至少支持：

- fixed
- per_area
- per_qty
- fixed_plus_qty
- fixed_plus_area

验收标准：

- 覆膜、烫金、UV、模切、打样可配置。
- 不同工艺可用不同计费方式。
- 报价明细能展示每项工艺成本。

### 7.4 报价规则管理

功能：

- 规则集列表。
- 规则集版本号。
- 适用客户类型。
- 优先级。
- 生效时间。
- 条件 JSON。
- 配置 JSON。

规则设计方式：

```text
规则结构化存储 + 后端代码解释执行
```

禁止把任意公式文本存入数据库后动态执行。

验收标准：

- 同一个模板可有多个规则集。
- 规则有版本号。
- 新规则只影响新报价。
- 旧报价可通过快照追溯。

## 16. 开发团队开工前确认清单

1. 第一阶段是否只做微信小程序 + H5。
2. 后台采用 React Admin 还是 Vue Admin。
3. 是否需要企业客户专属报价规则。
4. 是否需要不同业务员维护不同客户价格。
5. 材料成本单位是否统一按平方米。
6. 印刷费用是否全部按数量计费。
7. 工艺费用是否需要按面积、数量、固定费混合计算。
8. 模切是否统一包含刀模费。
9. 是否有税率单独计价要求。
10. 运费是否第一期独立规则计算。
11. 是否第一期支持 PDF 报价单导出。
12. 是否第一期支持询盘流转。
13. 库存第一期是否只做原材料库存。
14. 是否需要批次管理。
15. 是否需要微信消息或短信通知。
16. 是否需要多管理员角色权限。
17. 是否需要操作日志导出。
18. 后续是否计划对接 ERP、财务或生产系统。

---

## 17. 推荐执行顺序

严格建议按以下顺序推进：

```text
数据库设计
-> 后端基础模块
-> 后端报价引擎
-> 报价快照
-> 后台价格与规则配置
-> 用户端产品展示
-> 用户端在线报价
-> 历史报价
-> 操作日志
-> 基础库存
-> 联调测试
-> 部署验收
```

这样可以先锁定系统最核心的“报价可信度”，再逐步扩展展示、会员、库存和运营能力。
